Use tabs to group your properties in way that improves the editor experience.

Tabs

  • We had them in Umbraco until 8.0.0
  • Were replaced by groups
  • Made possible by community package Matryoshka
  • Back in Umbraco since 8.17.0

No tabs means a lot of scrolling

Not clear to editors where something can be found.

Tabs

  • Group them by functionality
  • Use groups to combine them in side a tab
  • Order tabs so frequently edited fields come first
  • Observe your editors to optimize structure.

Group by function => all meta data props in one tab

Group inside tab => seo and open graph

Order : your main content fields should be the first tab.

Mandatory properties

  • As a developer seems logical
  • But it can prevent editors from rapidly publishing content
  • Handle empty properties with a fallback
  • Only make them mandatory when needed

Developer => no need to handle empty properties

Editor => Hypothetical case

Fallbacks => Defensive programming, seo title to name, image to a default image or backgroungd color

Tweaking the backoffice

By using EditorModel Notifications

Which allow us to manipulate the model used by the backoffice before it is loaded into an editor.

Disable preview

						
							public class SendingContentNotificationHandler : INotificationHandler
							{																
								public void Handle(SendingContentNotification notification)
								{
									if (ContentTypes.Contains(notification.Content.ContentTypeAlias))
									{
										return;
									}
							
									notification.Content.AllowPreview = false;
																	
									var urls = new List();							
									var urlCultures = notification.Content.Urls.Select(x => x.Culture).Distinct().ToList();
							
									foreach (var culture in urlCultures)
									{
										urls.Add(UrlInfo.Message("This page has no links", culture));
									}
							
									notification.Content.Urls = urls.ToArray();
								}
							}
